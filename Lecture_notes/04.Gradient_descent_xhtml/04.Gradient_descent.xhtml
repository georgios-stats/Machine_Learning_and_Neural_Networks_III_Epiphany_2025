1 Motivations


Problem 1. Consider a learning problem ( H , Z , ℓ ) . Learning may involve the computation of the minimizer h * ∈ H , where H is a class of hypotheses, of the empirical risk function (ERF) R ˆ ( h ) = 1 n ∑ i = 1 n ℓ ( h , z i ) given a finite sample { z i ; i = 1 , ... , n } generated from the data generating model g ( ⋅ ) and using loss ℓ ( ⋅ ) ; that is h * = arg min h ∈ H ( R ˆ ( h ) ) = arg min h ∈ H ( 1 n ∑ i = 1 n ℓ ( h , z i ) ) (1.1) or its corresponding regularized version. If analytical minimization of (1.1) is impossible or impractical, numerical procedures can be applied; eg Gradient Descent (GD) algorithms. Such approaches introduce numerical errors in the solution.

2 subgradient

Definition 2. Vector v is called subgradient of a function f : S → R at w ∈ S if ∀ u ∈ S , f ( u ) ≥ f ( w ) + ⟨ u - w , v ⟩ (2.1)

Note 3. There may be more than one subgradients of a function at a specific point. As seen by (2.1), subgradients are the slopes of all the lines passing through the point ( w , f ( w ) ) and been under the function f ( ⋅ ) .

Definition 4. The set of subgradients of function f : S → R at w ∈ S is denoted by ∂ f ( w ) .

Fact 5. Properties of subgradient sets useful for the subgradient construction
  1. If function f : S → R is differentiable at w then the only subgradient of f at w is the gradient ∇ f ( w ) , and (2.1) is equality; i.e. ∂ f ( w ) = { ∇ f ( w ) } .
  2. for constants α , β and convex function f ( ⋅ ) , it is ∂ ( α f ( w ) + β ) = α ( ∂ f ( w ) ) = { α v : v ∈ ∂ f ( w ) , }
  3. for convex functions f ( ⋅ ) and g ( ⋅ ) , it is ∂ ( f ( w ) + g ( w ) ) = ∂ f ( w ) + ∂ g ( w ) = { v + u : v ∈ ∂ f ( w ) , and u ∈ ∂ g ( w ) }

3 Description of Gradient Descent


Problem 7. For the sake of notation simplicity and generalization, we will present Gradient Descent (GD) in the following minimization problem w * = arg min w ∈ H ( f ( w ) ) (3.1) where here f : R d → R , and w ∈ H ⊆ R d ; f ( ⋅ ) is the function to be minimized, e.g., f ( ⋅ ) can be an empirical risk function R ˆ ( ⋅ ) .

Algorithm 8. Gradient Descent (GD) algorithm with learning rate η t > 0 for the solution of the minimization problem (3.1)
For t = 1 , 2 , 3 , ... iterate:
  1. compute w ( t + 1 ) = w ( t ) - η t v t ; where v t ∈ ∂ f ( w ( t ) ) (3.2)
  2. terminate if a termination criterion is satisfied, e.g. If t ≥ T max then STOP

Note 9. (Intuition) Assume f is differentiable. GD produces a chain { w ( t ) } that drifts towards a minimum w * . The chain is directed towards the opposite direction of that of the subgradient ∇ f ( ⋅ ) and at a rate controlled by the learning rate η t .

Note 10. (More intuition) Assume f is differentiable. Consider the (1st order) Taylor polynomial for the approximation of f ( w ) in a small area around u (i.e. ∥ v - u ∥ = small ) f ( u ) ≈ P ( u ) = f ( w ) + ⟨ u - w , ∇ f ( w ) ⟩ Assuming convexity for f , it is f ( u ) ≥ f ( w ) + ⟨ u - w , ∇ f ( w ) ⟩ ⏟ = P ( u ; w ) (3.3) meaning that P lower bounds f . Hence we could design an updating mechanism producing w ( t + 1 ) which is nearby w ( t ) (small steps) and which minimize the linear approximation P ( w ) of f ( w ) at w ( t ) P ( w ; w ( t ) ) = f ( w ( t ) ) + ⟨ w - w ( t ) , ∇ f ( w ( t ) ) ⟩ . (3.4) while hoping that this mechanism would push the produced chain { w ( t ) } towards the minimum because of (3.3). Hence we could recursively minimize the linear approximation (3.4) and the distance between the current state w ( t ) and the next w value to produce w ( t + 1 ) ; namely w ( t + 1 ) = arg min ∀ w ( 1 2 ∥ w - w ( t ) ∥ 2 + η P ( w ; w ( t ) ) ) (3.5) = arg min ∀ w ( 1 2 ∥ w - w ( t ) ∥ 2 + η ( f ( w ( t ) ) + ⟨ w - w ( t ) , ∇ f ( w ( t + 1 ) ) ⟩ ) ) = w ( t ) - η ∇ f ( w ( t ) ) where parameter η > 0 controls the trade off in (3.5).

Note 11. Given T iterations of GD algorithm, the output of GD can be (but not a exclusively),
  1. the average (after discarding the first few iterations of w ( t ) for stability reasons) w GD ( T ) = 1 T ∑ t = 1 T w ( t ) (3.6)
  2. or the best value discovered w GD ( T ) = arg min ∀ w t ( f ( w ( t ) ) )
  3. or the last value discovered w GD ( T ) = w ( T )

Note 12. GD output (e.g. in Note 11) converges to a local minimum, w GD ( T ) → w * (in some sense), under different sets of regularity conditions (some are weaker other stronger). Section 4 has a brief analysis.

Note 13. The parameter η t is called learning rate (or step size, gain). It determines the size of the steps GD takes to reach a (local) minimum. { η t } is a non-negative sequence and it is chosen by the practitioner. In principle, regularity conditions (Note 12) often imply restrictions on the decay of { η t } which guide the practitioner to parametrize it properly. Some popular choices of learning rate η t are:
  1. constant; η t = η , for where η > 0 is a small value. The rationale is that GD chain { w t } performs constant small steps towards the (local) minimum w * and then oscillate around it.
  2. decreasing and converging to zero; η t ssearrow with lim t → ∞ η t = 0 . E.g. η t = ( C t ) ς where ς ∈ [ 0.5 , 1 ] and C > 0 . The rationale is that GD algorithm starts by performing larger steps (controlled by C ) at the beginning to explore the area for discovering possible minima. Also it reduces the size of those steps with the iterations (controled by ς ) such that eventually when the chain { w t } is close to a possible minimum w * value to converge and do not overshoot.
  3. decreasing and converging to a tiny value τ * ; η t ssearrow with lim t → ∞ η t = τ * E.g. η t = ( C t ) ς + τ * with ς ∈ ( 0.5 , 1 ] , C > 0 , and τ * ≈ 0 . Same as previously, but the algorithm aims at oscillating around the detected local minimum.
  4. constant until an iteration T 0 and then decreasing; Eg η t = ( C max ( t , T 0 ) ) ς with ς ∈ [ 0.5 , 1 ] and C > 0 , and T 0 < T . The rationale is that at the first stage of the iterations (when t ≤ T 0 ) the algorithm may need a constant large steps for a significant number of iterations T 0 in order to explore the domain; and hence in order for the chain { w t } to reach the area around the (local) minimum w * . In the second stage, hoping that the chain { w t } may be in close proximity to the (local) minimum w * the algorithm progressively performs smaller steps to converge towards the minimum w * . The first stage ( t ≤ T 0 ) is called burn-in; the values { w t } produced during the burn-in ( t ≤ T 0 ) are are often discarded/ignored from the output of the GD algorithm.
  • Parameters C , ς , τ * , T 0 may be chosen based on pilot runs against a small fraction of the training data set.

Note 14. There are several practical termination criteria that can be used in GD Algorithm 8(step 2.). They aim to terminate the recursion in practice. Some popular termination criteria are
  1. terminate when the gradient is sufficiently close to zero; i.e. if ∥ ∇ f ( w ( t ) ) ∥ ≤ ϵ , for some pre-specified tiny ϵ > 0 then STOP
  2. terminate when the chain w ( t ) does not change; i.e. if ∥ w ( t + 1 ) - w ( t ) ∥ ≤ ϵ ∥ w ( t ) ∥ for some pre-specified tiny ϵ > 0 then STOP
  3. terminate when a pre-specified number of iterations T is performed; i.e. if t ≥ T then STOP
Here (1) may be deceive if the chain is in a flat area, (2) may be deceived if the learning rate become too small, (3) is obviously a last resort.
